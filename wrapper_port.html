Wrapper Port¶

Class WrapperPort is an extension to nMigen’s Signal. It wraps a port, adding a new name and optionally slicing the signal. It adds these attributes:

WrapperPort.internal_name     # name of the port in internal source to be wrapped
WrapperPort.direction         # DIR_FANIN, DIR_FANOUT or DIR_NONE
WrapperPort.interface_name    # name of the group of ports (interface)
WrapperPort.bounds            # range of bits that belong to the port
                              # and range which is sliced from the port

See Port slicing to know more about bounds.

This is used in IPWrapper class implementation and there should be no need to use WrapperPort individually.

class WrapperPort(Signal)¶
__init__(self, bounds, name, internal_name, direction, interface_name)¶
Parameters

  • bounds[0:1] – upper and lower bounds of reference signal

  • bounds[2:3] – upper and lower bounds of internal port, which are either the same as reference port, or a slice of the reference port

  • name – a new name for the signal

  • internal_name – name of the port to be wrapped/sliced

  • direction – one of nmigen.hdl.rec.Direction, e.g. DIR_FANOUT
